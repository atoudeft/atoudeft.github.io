La référence this

Considérons la classe CompteBancaire suivante :

public class CompteBancaire {

private String numero;

private double solde;

public void crediter(double montant) {

solde = solde + montant;

}

}

Une méthode d’instance est appelée pour un objet donné. Par exemple, avec 2 objets de type CompteBancaire :

CompteBancaire cpt1 = new CompteBancaire(),

cpt2 = new CompteBancaire();

L’appel :

cpt1.crediter(200);

exécute la méthode crediter() pour l’objet cpt1, alors que l’appel :

cpt2.crediter(200);

exécute la même méthode mais cette fois-ci pour l’objet cpt2.

Dans le premier appel, c’est le solde de cpt1 qui est augmenté de 200 et dans le second appel c’est le solde de cpt2 qui est augmenté de 200.

Une méthode d’instance possède donc un paramètre implicite qui est l’objet qui lui fait appel. L’en-tête de la méthode crediter() montre un seul paramètre :

public void crediter(double montant)

Mais, en réalité, il y en a 2, comme on le voit dans l’appel :

cpt1.crediter(200);

paramètre explicite

paramètre implicite

Il arrive qu’on ait besoin d’accéder au paramètre implicite (l’objet qui fait appel à la méthode). En Java, cet objet est référencé par la référence prédéfinie this. On aurait donc pu définir la méthode crediter() comme ceci :

public void crediter(double montant) {

this.solde = this.solde + montant;

}

L’utilisation de this est nécessaire lorsque le paramètre d’une méthode ou d’un constructeur porte le même nom que l’attribut de l’objet, comme dans le constructeur et le mutateur du numéro de compte suivants :

public class CompteBancaire {

private String numero;

private double solde;

public CompteBancaire(String numero) {

this.numero = numero; //Ici this est nécessaire

this.solde = 0; //Ici this n’est pas nécessaire

}

public void setNumero(String numero) {

this.numero = numero; //Ici this est nécessaire

}

public void crediter(double montant) {

this.solde = this.solde + montant; //Ici this n’est pas nécessaire

}

}

Il arrive aussi qu’on utilise this pour faire référence à l’objet dans son entièreté, comme dans la méthode afficher() suivante qui affiche à l’écran la conversion en String du compte bancaire :

public class CompteBancaire {

private String numero;

private double solde;

public CompteBancaire(String numero) {

this.numero = numero; //Ici this est nécessaire

this.solde = 0; //Ici this n’est pas nécessaire

}

@Override

public String toString() {

return "Compte numéro [" + numero + "] avec " + solde + "$";

}

public void afficher() {

System.out.println(this);

}

}
