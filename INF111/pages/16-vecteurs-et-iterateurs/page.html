Utilisation des vecteurs

Les tableaux peuvent être créés dynamiquement (c'est-à-dire, leur taille peut être fixée à l’exécution) mais, une fois créé, la taille du tableau ne peut plus changer.

Pour avoir une structure de données similaire à un tableau mais dont la taille augmente au besoin, on peut utiliser des vecteurs.

Il existe plusieurs classes qui permettent de créer des vecteurs. Parmi ces classes, ArrayList est une des plus utilisées.

Un ArrayList encapsule un tableau dans lequel sont stockées les données insérées dans le ArrayList. Lorsque le tableau est plein, le ArrayList l’agrandit automatiquement.

On manipule un ArrayList à l’aide de ses méthodes telles que add(), remove(), get(),… On ne peut pas utiliser l’opérateur [] pour les vecteurs.

Remarque : classe Vector

La classe Vector est très similaire à ArrayList (fournissent les mêmes méthodes aux programmeurs et sont implémentées de la même façon à l’interne). La différence entre les 2 réside dans leur comportement dans un contexte multithread, c’est-à-dire en « programmation parallèle » (lorsque les instances sont manipulées par plusieurs méthodes en même temps).

Lorsqu’on n’est pas en contexte multithread, ArrayList est préférable à Vector.

Utilisation des vecteurs

À partir de Java 5, la classe ArrayList (et beaucoup d’autres classes similaires) a subi d’importantes modifications. ArrayList est devenue une classe générique, c'est-à-dire une classe paramétrée par un type.

En effet, même si on peut stocker dans un vecteur des objets de différents types (par exemple, mélanger des comptes bancaires, des dés et des String dans un même vecteur), cela n’arrive généralement pas en pratique. Un vecteur stocke habituellement des objets de la même famille (uniquement des comptes bancaires ou uniquement des dés ou uniquement des String mais pas de mélange). À partir de Java 5, le programmeur a donc la possibilité (c’est même très recommandé) d’indiquer, à la déclaration du vecteur, quel type d’éléments il a l’intention de stocker dans le vecteur. Cela permet au compilateur d’automatiser un certain nombre de tâches qui, avant Java 5, revenaient au programmeur.

À la déclaration d’une variable de type ArrayList, on spécifie donc le type des objets qu’on y stocke :

ArrayList<CompteBancaire> v1;

Et à l’instanciation :

v1 = new ArrayList<>();

On ne peut stocker dans v1 que des objets de type CompteBancaire :

CompteBancaire cpt1 = new CompteBancaire();

v1.add(cpt1); //OK

v1.add(new CompteBancaire()); //OK

v1.add("Hello"); //NON

Pour créer un vecteur qui stocke des données primitives, on utilise les classes enveloppes. Voici un vecteur destiné à contenir des entiers :

Vector<Integer> v2 = new Vector<>();

On peut insérer dans v2 des entiers :

v2.add(new Integer(48));

v2.add(Integer.valueOf(77));

On peut insérer une donnée primitive :

v2.add(37);

Le compilateur s’occupe d’envelopper l’entier 37 dans un Integer (opération appelée auto boxing).

Un des avantages des classes génériques est que le compilateur effectue certaines conversions automatiquement. Par exemple, la méthode get() de v1 nous retourne automatiquement une référence de type CompteBancaire :

CompteBancaire cpt = v1.get(0);

Remarque : avant Java 5

Avant Java 5, les classes telle que ArrayList n’étaient pas génériques. On ne spécifie donc pas le type d’objets qu’on stocke dans le vecteur :

ArrayList v = new ArrayList();

Ce qui fait qu’on peut stocker n’importe quel objet dans le vecteur. On peut même mélanger différents types d’objets (ce qui n’est pas réaliste) :

v.add(new CompteBancaire());

v.add(new Integer(59));

v.add("Bonjour");

Par contre, en accédant aux éléments du vecteur, on obtient des références de type Object qu’on doit transtyper vers le bon type :

CompteBancaire c = (CompteBancaire)v.get(0);

Si l’élément récupéré n’est pas du bon type, une exception de type ClassCastException est déclenchée. On doit donc s’en prémunir par une vérification avec l’opérateur instanceof :

Object x = v.get(0);

if (x instanceof CompteBancaire) {

c = (CompteBancaire)x;

}

Cette façon d’utiliser les vecteurs est maintenant déconseillée et il faut toujours utiliser les versions génériques de ces classes.

Itérateurs

Pour parcourir tous les éléments d’un vecteur, on peut utiliser une boucle for classique :

ArrayList<CompteBancaire> v1 = new ArrayList<>();

//…

int taille = v1.size();

for (int i=0;i<taille;i++) {

System.out.println("Compte numéro "+v1.get(i).getNumero()

+"; Solde : "+v1.get(i).getSolde()+"$");

}

Cette façon de faire n’est pas très efficace car la méthode get() ne mémorise pas la position de parcours d’une itération à l’autre.

Un itérateur permet de parcourir un vecteur (ou n’importe quelle collection) en gardant la position de parcours afin de trouver rapidement l’élément suivant à l’itération suivante. Voici comment on peut parcourir un vecteur v1 avec un itérateur :

ArrayList<CompteBancaire> v1 = new ArrayList<>();

//…

ListIterator<CompteBancaire> iterateur = v1.listIterator();

CompteBancaire cpt;

while (iterateur.hasNext()) {

cpt = iterateur.next();

System.out.println("Compte numéro "+cpt.getNumero()

+"; Solde : "+cpt.getSolde()+"$");

}

On n’instancie pas nous-même un itérateur. C’est le vecteur qui nous le fournit à l’aide de sa méthode listIterator().

À sa création, l’itérateur se place avant le premier élément. Il faut appeler next() pour obtenir l’élément suivant du vecteur, à commencer par le premier :

S’il n’y a pas d’élément suivant, l’appel à next() déclenche une exception de type NoSuchElementException. Il faut donc toujours vérifier l’existence de l’élément suivant avant d’appeler next(). On utilise pour cela la méthode hasNext().

Attention : chaque appel à next() parcourt un élément du vecteur et déplace l’itérateur à l’élément suivant. Il faut donc appeler une seule fois next() par itération. Le code suivant est erroné et dangereux :

ArrayList<CompteBancaire> v1 = new ArrayList<>();

//…

ListIterator<CompteBancaire> iterateur = v1.listIterator();

while (iterateur.hasNext()) {

System.out.println("Compte numéro "+ iterateur.next().getNumero()

+"; Solde : "+ iterateur.next().getSolde()+"$");

}

For each

On peut aussi parcourir un vecteur (ou un tableau) à l’aide d’une structure for each. À l’instar de l’itérateur, la structure garde la position courante grâce à une variable d’itération :

ArrayList<CompteBancaire> v1 = new ArrayList<>();

//…

for (CompteBancaire cpt : v1) {

System.out.println("Compte numéro "+cpt.getNumero()

+"; Solde : "+cpt.getSolde()+"$");
