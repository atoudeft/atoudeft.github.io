Introduction

Dans la plupart des langages, les chaînes de caractères sont représentées par un type intégré au langage. Ce n’est pas le cas en Java qui propose plutôt des classes dans l’API. La classe la plus importante est sans aucun doute la classe String du package java.lang.

Dans un programme java, une chaîne de caractères est un objet instance de la classe String. On crée cette instance en utilisant un constructeur parmi la dizaine offerte par la classe. Mais, on peut aussi utiliser les guillemets (double-quotes) qui constituent, en quelque sorte, un autre constructeur de chaînes. Voici des exemples de construction de chaînes :

String messageDeBienvenue = new String("Bonjour et bienvenue") ;

String prenom = "Bernard" ;

char[] t = {’M’,’o’,’n’,’s’,’i’,’e’,’u’,’r’};

String titre = new String(t);

Longueur d’une chaîne

Pour connaître la longueur d’une chaîne, on utilise la méthode length() :

String prenom = "Bernard" ;

int longueur = prenom.length() ;

System.out.println("La longueur du prénom est : "+longueur);

Extraction de caractères et de sous-chaînes

Pour connaître la valeur du caractère se trouvant à une position donnée de la chaîne, on utilise la méthode charAt() :

String prenom = "Bernard" ;

int i = 4 ;

char c = prenom.charAt(i) ;

System.out.println("Le caractère de la position "+i+" est: "+c);

La première position porte le numéro 0. La valeur de i doit être dans le bon intervalle (supérieure ou égale à 0 et inférieure strictement à la longueur de la chaîne) sinon une exception est déclenchée.

La méthode substring() permet d’extraire une sous-chaîne d’une chaîne existante s. L’instruction :

String r = s.substring(6) ;

Permet d’extraire tous les caractères situés à partir de la position 6. Tandis que l’instruction :

String r = s.substring(5,10) ;

Permet d’extraire tous les caractères situés entre les positions 5 et 9, inclusivement.

Conversions

1. Conversion de données primitives en chaînes de caractères

La classe String fournit la méthode statique valueOf(), surchargée plusieurs fois (ça veut dire qu’elle existe en plusieurs versions), pour convertir des données primitives en chaînes de caractères. Voici quelques exemples :

int x = 37 ;

String s1 = String.valueOf(x); //s1 est la chaîne "37"

double y = 2.36 ;

String s2 = String.valueOf(y); //s2 est la chaîne "2.36"

La méthode valueOf() fait le même travail que les méthodes statiques toString() des classes Integer, Double, Float,…. Ainsi, les appels à valueOf() précédents sont équivalents à :

String s1 = Integer.toString(x); //s1 est la chaîne "37"

String s2 = Double.toString(y); //s2 est la chaîne "2.36"

Une autre technique consiste à concaténer la donnée primitive avec une chaîne de caractères vide :

String s1 = ""+x; //s1 est la chaîne "37"

String s2 = ""+y; //s2 est la chaîne "2.36"

2. Conversion d’objets en chaînes de caractères

Les objets sont convertis en chaînes à l’aide de leur méthode toString(). La méthode toString() est automatiquement présente dans toute classe car elle est héritée de la classe Object dont dérive chaque classe. Cependant, la version de toString() définie dans la classe Object renvoie un résultat qui n’est pas toujours satisfaisant. Il convient donc de surdéfinir cette méthode dans nos classes. Par exemple :

public class Point

{

private double abscisse, ordonnee ;

public Point(double a, double o)

{

abscisse = a ;

ordonnee = o ;

}

public String toString()

{

return "("+abscisse+";"+ordonnee+")" ;

}

}

Cette méthode toString() est ensuite automatiquement invoquée lorsqu’il faut convertir un objet Point en chaîne de caractères :

Point p = new Point(5,-2.5) ;

System.out.println("Le point p est " + p) ;

3. Conversion de chaînes de caractères en données primitives

La conversion d’une chaîne de caractères en donnée primitive nécessite une analyse de la chaîne. Par exemple, pour convertir une chaîne de caractères en int, la classe Integer fournit la méthode parseInt(), pour convertir une chaîne de caractères en double, la classe Double fournit la méthode parseDouble(), et ainsi de suite :

String s1 = "235" ;

int x = Integer.parseInt(s1) ; //x vaut 235

String s2 = "29.13" ;

double y = Double.parseDouble(s2) ; //y vaut 29.13

4. Conversion de chaînes de caractères en majuscules/minuscules

Pour obtenir une chaîne de caractères qui contient les mêmes caractères qu’une chaîne s mais dont les minuscules sont changées en majuscules, on utilise la méthode toUpperCase() :

String s1 = "Bonjour" ;

String s2 = s1.toUpperCase(); //s2 vaut ”BONJOUR”.

Si on veut changer les majuscules en minuscules, on utilise la méthode toLowerCase() :

String s1 = "Bonjour" ;

String s3 = s1.toLowerCase(); //s3 vaut ”bonjour”.

3.5. Conversion de chaînes de caractères en tableaux de caractères :

La méthode toCharArray() retourne un tableau dont chaque case contient un caractère de la chaîne :

String s1 = "Bonjour" ;

char t[] = s1.toCharArray();//t vaut {’B’,’o’,’n’,’j’,’o’,’u’,’r’}.

Concaténation

Pour concaténer 2 chaînes de caractères, on utilise l’opérateur + :

String s1 = "Bonjour " ;

String s2 = "tout le monde" ;

String s3 = s1 + s2 ; //s3 est la chaîne "Bonjour tout le monde".

Lorsqu’on applique l’opérateur + à une chaîne et une donnée primitive, cette dernière est d’abord convertie en chaîne, puis les 2 chaînes sont concaténées. Ainsi, on peut calculer la surface d’un cercle d’un rayon donné :

int rayon = 10 ;

double surface = Math.PI* Math.pow(rayon,2);

Puis afficher le résultat dans un message :

System.out.println("La surface du cercle de rayon "+rayon+" est égal à " +surface);

Lorsque l’opérateur + est appliqué à une chaîne de caractères et un objet, ce dernier est converti en chaîne à l’aide de sa méthode toString(). Ensuite, les 2 chaînes sont concaténées.

Pour concaténer 2 chaînes de caractères, on peut aussi utiliser la méthode concat :

String s1 = "Bonjour " ;

String s2 = "tout le monde" ;

String s3 = s1.concat(s2);//donne le même résultat que s3 = s1+s2 ;

Comparaison

1. Test d’égalité

Pour savoir si 2 chaînes ont la même valeur, on utilise, comme pour tous les objets, la méthode equals(). L’opérateur == compare les références :

String s1 = new String("Bonjour") ;

String s2 = new String("Bonjour") ;

if (s1 == s2) //condition fausse

{

...

}

if (s1.equals(s2)) //condition vraie

{

...

}

La méthode equals() tient compte de la casse des caractères. Ainsi, les chaînes "Bonjour" et "bonjour" ne sont pas égales selon equals().

Pour comparer 2 chaînes indépendamment de la casse des caractères, on utilise la méthode equalsIgnoreCase().

String s1 = new String("BONJOUR") ;

String s2 = new String("bonjour") ;

if (s1.equals(s2)) //condition fausse

{

...

}

if (s1.equalsIgnoreCase(s2)) //condition vraie

{

...

}

2. Test d’inégalité

Chaque plate-forme utilise un système de codage pour représenter les caractères. Le code ASCII et le jeu de caractères Unicode sont des exemples de systèmes de codage. Ces systèmes de codage définissent un ordre sur les caractères : un caractère est supérieur à un autre si son code est plus grand.

L’ordre sur les caractères induit un ordre sur les chaînes de caractères. L’ordre entre 2 chaînes est déterminé par l’ordre entre leurs premiers caractères, s’ils sont différents. Sinon, par leurs deuxièmes caractères, s’ils sont différents. Sinon, par leurs troisièmes, et ainsi de suite. En résumé, l’ordre entre 2 chaînes est déterminé par l’ordre entre le premier caractère qui les différencie.

Par exemple, dans le système de codage ASCII, la chaîne "bonjour" est inférieure à la chaîne "bonsoir" car le code du caractère ‘j’ est inférieur au code du caractère ‘s’.

Les méthodes qui déterminent l’ordre sur les chaînes retournent un entier. Cet entier a la valeur 0 si les 2 chaînes sont égales, une valeur négative si la première chaîne est inférieure à la seconde et une valeur positive si la première chaîne est supérieure à la seconde.

La méthode compareTo() compare 2 chaînes de caractères en tenant compte de leur casse, tandis que la méthode compareToIgnoreCase() les compare sans tenir compte de leur casse. Par exemple :

String s1 = new String("Bonjour") ;

String s2 = new String("Bonjour") ;

int r = s1.compareTo(s2);

if (r==0) //s1 et s2 sont égales

{

...

}

else

if (r<0) //s1 est inférieure à s2

{

...

}

else //s1 est supérieure à s2

{

...

}

3. Autres tests

Les méthodes startsWith() et endsWith() vérifient si une chaîne débute ou se termine, respectivement, par une sous-chaîne spécifique. Ainsi, l’appel :

s.startsWith(”bon”);

retourne true si la chaîne s commence par la chaîne "bon" et false dans le cas contraire. Tandis que l’appel :

s.startsWith(”bon”,9);

retourne true si la sous-chaîne de s commençant à partir du caractère d’indice 9 débute par la chaîne "bon" et false dans le cas contraire. Autrement dit, l’appel est identique à :

s.substring(9).startsWith(”bon”);

Et l’appel :

s.endsWith(”soir”);

retourne true si la chaîne s se termine par la chaîne "soir" et false dans le cas contraire.

Recherche

La méthode indexOf() vérifie si une chaîne ou un caractère est présent(e) dans une autre chaîne. Ainsi, l’appel :

int i = s.indexOf(”monde”);

retourne la position (l’indice) où se trouve le mot monde dans la chaîne s. Si le mot monde ne se trouve pas dans la chaîne s, la méthode retourne -1.

Si le mot monde est présent plusieurs fois dans la chaîne s, la méthode retourne la position de la première occurrence uniquement.

De même, l’appel :

int i = s.indexOf(’j’);

retourne la position (l’indice) où se trouve le caractère j dans la chaîne s. Si le caractère j ne se trouve pas dans la chaîne s, la méthode retourne -1.

Là aussi, si le caractère j est présent plusieurs fois dans la chaîne s, la méthode retourne la position de la première occurrence uniquement.

L’appel :

int i = s.indexOf(”monde”,10);

retourne la position (l’indice) où se trouve le mot monde dans la chaîne s mais commence la recherche à partir de la position 10. Si le mot monde n’est pas trouvé, la méthode retourne -1.

Il en est de même pour l’appel :

int i = s.indexOf(’j’,10);

La méthode lastIndexOf() fonctionne comme indexOf() mais retourne la position de la dernière occurrence de la chaîne ou du caractère. Ainsi, l’appel :

int i = s.lastIndexOf(”monde”);

retourne la position (l’indice) où se trouve la dernière occurrence du mot monde dans la chaîne s. Si le mot monde ne se trouve pas dans la chaîne s, la méthode retourne -1.

De même, l’appel :

int i = s.lastIndexOf(’j’);

retourne la position (l’indice) où se trouve la dernière occurrence du caractère j dans la chaîne s. Si le caractère j ne se trouve pas dans la chaîne s, la méthode retourne -1.

Analyse de chaînes : la méthode split()

La méthode split() découpe une chaîne en morceaux et retourne un tableau qui contient les morceaux à raison de 1 morceau par case. Les morceaux sont déterminés par le séparateur fourni à la méthode.

Voici des exemples :

Si on considère la chaîne s suivante :

String s = "Dupont,Paul,34:Tremblay,Serge,25:Thibaut,Annie,33" ;

Alors, l’instruction :

String[] t1 = s.split(":");

Produit un tableau ayant le contenu suivant :

t1[0] = Dupont,Paul,34

t1[1] = Tremblay,Serge,25

t1[2] = Thibaut,Annie,33

L’instruction :

String[] t2 = s.split(",");

Produit un tableau ayant le contenu suivant :

t2[0] = Dupont

t2[1] = Paul

t2[2] = 34:Tremblay

t2[3] = Serge

t2[4] = 25:Thibaut

t2[5] = Annie

t2[6] = 33

Et l’instruction :

String[] t3 = s.split("[:,]");

String premierMorceau = s.split("[:,]")[0];

Produit un tableau ayant le contenu suivant :

t3[0] = Dupont

t3[1] = Paul

t3[2] = 34

t3[3] = Tremblay

t3[4] = Serge

t3[5] = 25

t3[6] = Thibaut

t3[7] = Annie

t3[8] = 33

Divers

Il existe d’autres méthodes utilitaires sur les chaînes. En voici une liste non exhaustive :

  • trim() : supprime les espaces au début et à la fin de la chaîne. Voici un exemple :

String reponse = " oui ";

String rep = reponse.trim() ; //rep vaut "oui"

  • replace() : crée une copie d’une chaîne mais en prenant soin de remplacer toutes les occurrences d’un caractère par un autre caractère.

Par exemple, l’instruction :

String r = s.replace(’a’,’e’);

retourne dans r une chaîne identique à la chaîne s mais où les caractères a sont remplacés par des e.
