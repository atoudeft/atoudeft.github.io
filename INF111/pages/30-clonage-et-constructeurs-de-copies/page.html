Introduction

Java utilise des références pour manipuler les objets car c’est plus efficace que d’utiliser des valeurs : les opérations telles que l’affectation et la transmission et retour des objets à des méthodes se font plus rapidement et à moindre coût puisque les objets ne sont pas copiés (ce sont les adresses mémoires qui sont copiées).

Il arrive, cependant, qu’on veuille créer des copies d’objets. Le clonage et le constructeur de copies sont les deux mécanismes utilisés pour cela.

Clonage

Cloner un objet, c’est créer une copie identique de l’objet.

En Java, la classe Object fournit la méthode clone() dont hérite toute classe. On peut donc cloner n’importe quel objet en utilisant sa méthode clone(). Sauf que la méthode clone() de la classe Object n’est pas publique, elle est protégée (protected). Nous devons donc la redéfinir avec une version publique et faire appel à la méthode clone() de la classe Object en utilisant super.clone().

D’autre part, le clonage est désactivé par défaut. Pour l’activer, la classe doit implémenter l’interface Cloneable. Cloneable est une interface de marquage[1] (tagging interface), c’est-à-dire qu’elle ne contient aucune méthode. Son seul rôle est d’activer le clonage pour les objets de la classe.

Voici un exemple d’une classe Point, qui représente des points dans le plan 2D, pour laquelle nous avons activé le clonage et redéfini la méthode clone() :

public class Point implements Cloneable {

private double abscisse, ordonnee;

public Point() {

}

public Point(double abscisse, double ordonnee) {

this.abscisse = abscisse;

this.ordonnee = ordonnee;

}

public Point(double abscisse) {

this.abscisse = abscisse;

}

@Override

public Object clone() throws CloneNotSupportedException {

return super.clone();

}

//etc...

}

On peut maintenant cloner des objets de type Point :

public static void main(String[] args) {

Point p1 = new Point(5,8);

try {

Point p2 = (Point)p1.clone();

Ça fonctionne très bien pour notre classe Point. Cependant, la méthode clone() de la classe Object effectue des copies superficielles des objets. Et ce n’est pas toujours ce qu’on veut.

1. Copies superficielles

Faisons le même travail pour cloner des objets de type Cercle :

public class Cercle implements Cloneable {

private Point centre;

private double rayon;

public Cercle() {

centre = new Point();

rayon = 10;

}

public Cercle(Point c, double r) {

centre = c;

rayon = r;

}

@Override

public Object clone() throws CloneNotSupportedException {

return super.clone();

}

public Point getCentre() {

return centre;

}

//etc...

}

Lorsqu’on clone un cercle, la copie obtenue n’est pas parfaite (elle est superficielle) :

abscisse : 5

ordonnee : 8

c1

clonage

rayon : 20

c2

centre :

rayon : 20

public static void main(String[] args) {

Point p1 = new Point(5,8);

Cercle c1 = new Cercle(p1,20);

try {

Cercle c2 = (Cercle)c1.clone();

} catch (CloneNotSupportedException e) {

e.printStackTrace();

}

}

Ici, la méthode clone() copie les 2 attributs centre et rayon. Sauf que centre est une référence. C’est donc l’adresse mémoire qui est copiée et non pas l’objet Point référencé par centre. Le cercle et sa copie se partagent donc le même centre.

En réalité, la méthode clone() de la classe Object effectue le clonage en copiant les valeurs des attributs dans le clone. Cela fonctionne parfaitement lorsque les attributs sont de types primitifs ou des String (car les objets String sont inaltérables). Dès que au moins un des attributs n’est pas primitif ni String, il faut faire un effort supplémentaire pour créer des copies profondes.

2. Copies profondes

Une copie profonde est une copie complète de l’objet : l’objet et sa copie ne se partagent aucun objet en commun.

Pour réaliser des copies profondes, la méthode clone() de notre classe doit :

  1. Appeler super.clone() pour obtenir une copie superficielle;
  2. Cloner individuellement tous les attributs qui ne sont pas primitif ou de type String. Cela suppose que les objets attributs sont eux aussi clonables.

Voici la bonne version de la méthode clone() pour cloner des cercles (ça suppose qu’on peut cloner des points) :

public class Cercle implements Cloneable {

private Point centre;

private double rayon;

public Cercle() {

centre = new Point();

rayon = 10;

}

public Cercle(Point c, double r) {

centre = c;

rayon = r;

}

@Override

public Object clone() throws CloneNotSupportedException {

Cercle copie = (Cercle)super.clone();

copie.centre = (Point)this.centre.clone();

return copie;

}

//etc...

}

Constructeurs de copies

Un constructeur de copies réalise lui aussi des copies d’objets. Le principe est de transmettre à ce constructeur un objet qu’il s’occupe de copier dans l’instance qu’il crée.

Voici un exemple de constructeur de copies de la classe Point :

public class Point {

private double abscisse, ordonnee;

public Point() {

}

public Point(double abscisse, double ordonnee) {

this.abscisse = abscisse;

this.ordonnee = ordonnee;

}

public Point(double abscisse) {

this.abscisse = abscisse;

}

//Constructeur de copies :

public Point(Point original) {

this.abscisse = original.abscisse;

this.ordonnee = original.ordonnee;

}

//etc...

}

Qu’on peut utiliser pour créer des copies de points :

public static void main(String[] args) {

Point p1 = new Point(5,8);

Point p2 = new Point(p1);

}

Essayons de faire la même chose avec les cercles :

public class Cercle {

private Point centre;

private double rayon;

public Cercle() {

centre = new Point();

rayon = 10;

}

public Cercle(Point c, double r) {

centre = c;

rayon = r;

}

public Cercle(Cercle original) {

this.centre = original.centre; //(1)

this.centre = new Point(original.centre); //(2)

this.rayon = original.rayon;

}

//etc...

}

L’instruction (1) est barrée car elle est fausse. Elle ne copie que l’adresse mémoire du centre (la copie du cercle est donc superficielle). L’instruction doit être remplacée par l’instruction (2) qui crée une copie du centre à l’aide du constructeur de copies de la classe Point.

  1. Il existe quelques interfaces de marquage dans la bibliothèque Java. Cloneable et Serializable, utilisée dans les flux d’entrées/sorties, sont parmi les plus utilisées. Serializable sert à activer le mécanisme de sérialisation pour des objets. ↑
