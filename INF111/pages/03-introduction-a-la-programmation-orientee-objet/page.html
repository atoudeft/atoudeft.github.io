Introduction

Des objets, des objets, encore des objets …

Nous sommes entourés d’objets : tables, chaises, ordinateurs, stylos, voitures, immeubles, …

Nous-mêmes, comme personnes, nous sommes des objets.

Certains objets sont difficiles à voir ou à toucher : des molécules, des planètes, des ouragans, ….

D’autres encore n’ont pas d’existence réelle et palpable : des personnages de jeux vidéo, des formules mathématiques, des idées, des tâches, des comptes bancaires, des rendez-vous médicaux, …

Si on regarde de plus près, les objets sont décrits par des caractéristiques ou attributs : une voiture est caractérisée par la marque, le modèle, l’année de construction, la couleur, la puissance du moteur, etc… Un compte bancaire est caractérisé par son numéro, les informations sur son propriétaire, le solde, le type de compte, etc…

Les attributs décrivent les objets, ou plus exactement l’état des objets. La figure suivante montre des exemples d’objets avec leurs attributs :

Les objets possèdent un comportement : une voiture peut avancer, reculer, stationner, … Un compte bancaire peut être crédité, débité, fermé, … Un étudiant peut s’inscrire à des cours, passer un examen, imprimer son bulletin, …

Par leur comportement, les objets peuvent agir sur d’autres objets et affecter leur état : une personne peut faire avancer une voiture, un client peut créditer son compte, un enseignant peut attribuer une note à un étudiant pour un cours donné, etc…

De manière générale, les objets collaborent pour réaliser des tâches.

La programmation orientée objet s’inspire du monde réel. Un programme orienté objet est composé d’objets qui collaborent pour réaliser les fonctionnalités du programme. Chaque objet du programme possède son état et son comportement.

Classes

Un objet est donc une donnée structurée qui regroupe ensemble plusieurs données plus simples appelées attributs de l’objet.

Nous disposons déjà de types de données primitifs pour créer et manipuler des données simples ou élémentaires : nombres entiers, nombres réels, caractères et booléens.

Cependant, le langage de programmation, Java ou autre, ne fournit pas de types de données pour créer tous les objets possibles et imaginables. Nous devons créer nous-mêmes les types de données nécessaires pour créer et manipuler les objets dont on a besoin, tels que voitures, programmeurs, comptes bancaires, …

Les types de données qui permettent de créer des objets s’appellent des classes. Les classes sont des modèles qui décrivent la structure (c’est à dire les attributs) et le comportement des objets.

Ainsi, une classe Voiture va décrire la structure et le comportement des objets de type voiture, une classe CompteBancaire va décrire la structure et le comportement des objets de type compte bancaire, etc.

Voici les classes qui permettent de créer des objets tels que ceux décrits par la figure 1 :

//Fichier : Voiture.java

public class Voiture {

String numeroPlaque;

String marque;

String modele;

int annee;

double consommation;

double quantiteGazReservoir;

}

//Fichier : Point2D.java

public class Point2D {

double abscisse;

double ordonnee;

}

//Fichier : Programmeur.java

public class Programmeur {

String courriel;

String nom;

String langage;

}

//Fichier : CompteBancaire.java

public class CompteBancaire {

String numero;

String nomTitulaire;

double solde;

}

Notez-bien que chaque classe est publique (public) et est définie dans son propre fichier. Ce fichier doit porter le même nom que la classe, en respectant la casse des caractères.

Valeurs et références

On peut maintenant déclarer, dans un programme, des variables de ces classes comme, par exemple :

Voiture v1;

Cependant, la déclaration de cette variable ne crée pas d’objet. Cette variable est une référence destinée à contenir l’adresse mémoire d’un objet. C’est la principale distinction entre une variable d’un type primitif et une variable de type objet : une variable de type primitif contient la valeur de la donnée primitive alors qu’une variable de type objet contient l’adresse mémoire d’un objet.

Instanciation

La création d’un objet se fait par instanciation d’une classe. L’instanciation est le mécanisme qui permet de créer, à partir d’une classe, un objet de cette classe.

En Java, l’instanciation se fait à l’aide de l’opérateur new :

new Voiture();

Bien évidemment, il convient de référencer l’objet par une variable afin de pouvoir l’utiliser :

v1 = new Voiture();

L’adresse mémoire de l’objet créé est ainsi stockée dans la variable v1. On dit que v1 référence l’objet (ou que v1 pointe sur l’objet) :

La figure 3.a montre la représentation en mémoire d’un objet de type Voiture référencé par une variable v1. Les cases de la mémoire ont des numéros. Ces numéros sont des adresses mémoire. L’adresse de l’objet Voiture est l’adresse de la première case de l’espace qui lui a été alloué. Cette adresse est stockée dans la case allouée à la variable v1. C’est comme ça que la variable v1 référence l’objet Voiture. Notez que, par convenance, les adresses mémoire sont écrites dans le système hexadécimal (d’où le 0x devant l’adresse).

La représentation de la figure 3.a est fidèle de ce qui se passe en mémoire mais, on s’y perd rapidement dès qu’on a plusieurs variables et plusieurs objets à représenter. Lors de la mise au point d’un programme, on lui préfère souvent la représentation de la figure 3.b. La flèche qui va de la case de v1 vers l’objet signifie que v1 référence l’objet. Les valeurs réelles des adresses mémoire ne nous intéressent pas ici et n’ont aucune incidence sur notre code.

Comme pour les types de données primitifs, la classe permet au compilateur de connaître :

  1. La taille de l’espace mémoire alloué à un objet instance de la classe;
  2. Les valeurs possibles;
  3. Les opérations possibles.

Valeurs par défaut des attributs

En Java, les attributs d’un objet ont les valeurs suivantes par défaut :

  • 0, pour les attributs de type numérique (nombres entiers, nombres réels et caractères);
  • null, pour les attributs de type objet (y compris les tableaux et les chaînes de caractères);
  • false, pour les attributs de type booléen (boolean);

Accès aux attributs : l’opérateur point (.)

L’accès aux attributs d’un objet se fait à l’aide de l’opérateur point (.) selon la syntaxe :

objet.attribut

public static void main(String[] args) {

CompteBancaire cpt1 = new CompteBancaire();

cpt1.numero = "A1254DF65";

cpt1.titulaire = "Annie Dupont";

cpt1.solde = 2560;

cpt1.solde = cpt1.solde - 150;

System.out.println("Solde du compte : " + cpt1.solde + "$");

}

Encapsulation

La manipulation directe, dans un programme, des attributs d’un objet pose beaucoup de problèmes dont voici les deux principaux :

  1. Comment contrôler les opérations sur les attributs ?

Par exemple, comment empêcher d’affecter des valeurs négatives au solde d’un compte bancaire ? ou comment prélever systématiquement des frais d’opération de 1$ chaque fois que le solde du compte est inférieur à 1000$ ? …

Comment empêcher de modifier directement la quantité de gaz dans le réservoir d’une voiture, puisque cette dernière doit diminuer uniquement lorsque la voiture roule et augmenter uniquement lorsqu’on fait le plein ?

  1. Comment changer la structure des objets sans avoir à modifier tous les programmes qui les utilisent ?

La structure des objets est appelée à évoluer au fur et à mesure qu’on améliore les programmes et qu’on ajoute des fonctionnalités. Il est important de ne pas obliger les développeurs à modifier leur code à chaque fois qu’on modifie la structure des objets.

La solution à ces problèmes tient en un seul mot : encapsulation. C’est un concept fondamental de la programmation orientée objet.

L’encapsulation consiste simplement à rendre privés les attributs d’un objet. De cette façon, l’objet cache ses attributs au monde extérieur et il n’est plus possible d’y accéder dans un programme.

En Java, pour rendre un attribut privé, on utilise le mot-clé private :

//Fichier : CompteBancaire.java

public class CompteBancaire {

private String numero;

private String nomTitulaire;

private double solde;

}

Maintenant que les attributs sont privés, il n’est plus possible d’y accéder directement dans un programme :

public static void main(String[] args) {

CompteBancaire cpt1 = new CompteBancaire();

cpt1.numero = "A1254DF65";

cpt1.titulaire = "Annie Dupont";

cpt1.solde = 2560;

cpt1.solde = cpt1.solde - 150;

System.out.println("Solde du compte : " + cpt1.solde + "$");

}

Pour accéder aux attributs d’un objet en dehors de la classe, cette dernière doit fournir des méthodes d’accès.

Méthodes d’accès

En Java, pour rendre un attribut privé, on utilise le mot-clé private :

//Fichier : CompteBancaire.java

public class CompteBancaire {

private String numero;

private String nomTitulaire;

private double solde;

public void setNumero(String nouveauNumero) {

numero = nouveauNumero;

}

public void setTitulaire(String nouveauTitulaire) {

nomTitulaire = nouveauTitulaire;

}

public String getNomTitulaire() {

return nomTitulaire;

}

public double getSolde() {

return solde;

}

}
